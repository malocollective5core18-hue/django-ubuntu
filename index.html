System Update & Dependencies

Step: Prepare System

Commands:

sudo apt update
sudo apt install python3 python3-pip python3-venv -y

Description: Updates system packages and installs Python 3, pip (package manager), and venv (virtual environment tool).

Why needed: Ensures you have Python and necessary tools installed before creating a Django project.

Project & Directory Setup

Step: Create Project Directory

Commands:

mkdir my_django_project
cd my_django_project

Description: Creates a new directory for your Django project and enters it.

Why needed: Organizes all project files in one location for better project management.

Virtual Environment Setup

Step: Create Virtual Environment

Command:

python3 -m venv venv

Description: Creates an isolated Python environment for the project.

Why needed: Prevents package conflicts between projects. Each project has its own dependencies.

Note: The first "venv" is the module, the second is the environment name.

Activate Environment

Step: Activate Virtual Environment

Command:

source venv/bin/activate cd mysite

Description: Activates the virtual environment for your terminal session.

Why needed: Ensures packages are installed in the project environment, not system-wide.

Note: Your prompt will change to show (venv) when activated.

Install Django

Step: Install Django Package

Command:

pip install django

Description: Installs Django framework in the virtual environment.

Why needed: Django is the web framework needed to build the application.

Note: Only installs in the current venv, not system-wide.

Verify Django Installation

Step: Check Django Version

Command:

django-admin --version

Description: Displays the installed Django version.

Why needed: Confirms Django was installed successfully and shows which version.

Create Django Project

Step: Start New Django Project

Commands:

django-admin startproject mysite
cd mysite

Description: Creates a new Django project named "mysite" with default structure.

Why needed: Sets up the base project structure with configuration and settings files.

Note: This creates manage.py and settings folder automatically.

Database Migrations

Step: Apply Database Migrations

Command:

python manage.py migrate

Description: Applies database migrations to set up the database schema.

Why needed: Creates necessary database tables for Django's built-in features (auth, admin, etc.).

Note: Creates db.sqlite3 file in the project directory.

Run Development Server

Step: Start Development Server

Command:

python manage.py runserver

Description: Starts the lightweight development web server.

Why needed: Allows you to test your Django application locally.

Note: Access it at http://127.0.0.1:8000/ or http://localhost:8000/

Create Admin User

Step: Create Superuser

Command:

python manage.py createsuperuser

Description: Creates an admin user account for Django admin panel.

Why needed: Allows you to access the Django admin interface at /admin/

Note: You'll be prompted for username, email, and password.

Deactivate Environment

Step: Exit Virtual Environment

Command:

deactivate

Description: Exits the virtual environment and returns to system Python.

Why needed: Cleans up terminal when done with development.

Note: The (venv) prefix will disappear from your prompt.

Creating a New Project

Alternative Path: Start Fresh Project

Command:

creating a new project

Description: Indicates the decision point to create a brand new Django project.

Why needed: Shows the branching path for starting a new project vs. working with existing one.

Turning Off the Environment

Step: Shutdown Process

Command:

Turning off the environment

Description: Indicates the process of shutting down the Django development environment.

Why needed: Shows the proper way to cleanly exit the development session.

Stop Server

Step: Stop Development Server

Command:

CTRL + C

Description: Keyboard shortcut to stop the running Django development server.

Why needed: Gracefully stops the server process running on localhost:8000.

Note: This interrupts the running process and returns to command prompt.

Turning On the Environment

Step: Reactivate Virtual Environment

Command:

Turning on the environment

Description: Indicates reactivating the virtual environment for continued development.

Why needed: Shows how to resume work in the same project environment.

Navigate to Project

Step: Change Directory

Command:

cd my_django_project

Description: Navigate to the Django project directory.

Why needed: Ensures you're in the correct directory to run Django commands.

Note: This assumes the project directory exists from previous setup.

Reactivate Virtual Environment

Step: Activate Virtual Environment

Command:

source venv/bin/activate

Description: Reactivates the virtual environment in the project directory.

Why needed: Ensures you're using the project's isolated Python environment.

Note: Your prompt will show (venv) when activated.
